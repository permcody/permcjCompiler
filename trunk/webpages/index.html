 5/7/04 It has come up twice now that getopt on Sun and on Linux work slightly differently... probably no surprise. Here is how: getopt on Linux will look though all of argv for any dash options. On Sun it looks for the first contiguous set of dash options. Any nondash option like a filename disrupts it the search. The result is that: 

c- factor.c- -o factor.tm
may work on Linux but not on Sun. Not trying to be tricky but just writing the tests up I coded all the tests with the -o at the end. If you see your TM code in the test output, or you seen lots of TM usage messages, or you find that all your *.tm files are missing this might be the cause.

Not a problem a simple enclosing loop about the getopt function and an increment of optind (see man page) whenever you grab a filename will get past this on Sun.

Class Description

An introduction to compiler design and construction. This is a hands-on construction course in which each student will work independently to construct a compiler that actually works.

Text: Compiler Construction: Principles and Practice by
Kenneth Louden, ISBN: 0534939724, Published by Brooks and Cole

CRN:41013,53781,53782
Time: 9:30-11:20 TTh
Location: Engineering Physics 203

Class Goals

  1. Understand the basic theory of languages and techniques for simple language translation.
  2. Understand the features of compilers and interpreters.
  3. Understand the basic components and layered design of a compiler and the rational for their use.
  4. Each student will be responsible for independently building a simple compiler for a virtual machine and augment it with new features. The instructor will help the student to achieve this goal over a series of assignments, but it is up to the student to keep up with the assignments and complete each on time.
