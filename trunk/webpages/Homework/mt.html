  1. Name the six parts of a compiler on page 7 in your book and the products of each part.

  2. What is the purpose of generating tokens?

    1. lexical analysis error.

  6. What 4 operations are allowed in basic regular expressions. (p38)

  7. Write a regular expression suitable for flex for the following or tell me why it cannot be done.
    1. All strings of digits which do not begin with a 0 but allow the number 0.

    2. A list of lowercase letters separated by commas. (e.g. x,y,x)

    3. A list of alternating a's and b's (e.g. ababa or ba or bab or b)

    4. A list containing unnested matched pairs of parentheses

    5. A list containing nested matched pairs of parentheses

    6. A list of 3 a's followed by 3 b's

    7. A list of k a's followed by k b's

  8. Draw a NFA for (a|b)*a(a|b|EPSILON). Convert it to a DFA by collapsing the epsilon productions. See around page 66.

  9. Why is a context free grammar more powerful than a regular grammar?

  10. Consider this grammar: 
    stmt -> IF ( test ) stmt ENDIF
stmt -> IF ( test ) stmt ELSE stmt ENDIF
stmt -> ACTION 
stmt -> compound-stmt
compound-stmt -> { stmt-list }
stmt-list -> stmt | stmt ; stmt-list
test -> EXP
    and consider the statement IF ( EXP ) ACTION ELSE { ACTION ; ACTION }
    1. Draw the parse tree for this statement.

    2. Draw the syntax tree for this statement.

    3. Draw the treeNode data structure from our C- compiler for this statement.

  11. In class we used a trick for solving the dangling else problem that involved splitting the if statement into two classes of if statements one called matched and the other called unmatched. Write out the grammar for the matched/unmatched solution from class for the C- if statement using Bison style formating. Do not do the while statement. Do not include the action part of the grammar. Begin with the start symbol statement and assume the only other statements are return statments which look like RETURN exp ; and that you have an nonterm exp you can use in the test position.

    Now add an unless statement: unless ( test ) statement and a loop forever statement: loop statement forever You can have statements statement if exp ; and if ( exp ) { statement }. Perl requires the second if to have braces. Why? Using only these two kinds of statements give an example where not having the braces would be a problem.
